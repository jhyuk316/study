# 기타

## Private 메소드 TEST

- 웬만하면 private 메소드를 테스트하지 말 것
- private 메소드를 테스트하는 코드를 작성하는 것은 뭔가 좋지 못한 신호
- private 메소드의 테스트 코드를 작성하기 보다는 현재 코드 구조나 상태 등을 확인해볼 것

> 아마도 public을 위주로 private 메서드에 걸리는 테스트 케이스를 통해 검증하라는 뜻 같음.
